### 问题介绍 (0:15-0:45)

```
多模态大语言模型（VLM）在处理图表时，经常出现数字失认症现象。

比如：图表显示苹果销量是42，它却说成55；
或者把 A 产品的最高销量安到 B 产品上；
再或者完全忽略图表中的异常数据点。

这些错误看似小问题，但在数据分析、医疗报告、金融图表等场景中，可能造成严重后果。
```

**画面**：展示几种典型错误案例

### 解决方案 (0:45-1:30)

```
我们的解决方案是：合成数据 + LoRA 微调。

第一步：用 Python 自动生成各种类型的图表（柱状图、折线图、表格），并保存精确的元数据作为标准答案。

第二步：模拟 VLM 的错误行为，生成大量带错误标注的训练数据。

第三步：使用 LoRA 技术，微调 Qwen-VL-Chat 模型，只训练 0.1% 的参数。

这样，我们就得到了一个专门的事实核查模型。
```

## 录制技巧

### 1. 分段录制

将视频分成 4-5 段录制，避免一次性录制出错：
1. 开场 + 问题介绍
2. 解决方案介绍
3. 效果展示（录屏）
4. 结尾

### 2. 语速控制

- 保持稳定的语速（约 150 字/分钟）
- 关键数据停顿一下
- 避免口头禅

### 3. 画面质量

- 分辨率：1920x1080
- 帧率：30fps
- 鼠标点击时有高亮效果

### 4. 后期处理

- 添加字幕（可使用剪映自动生成）
- 添加背景音乐（音量低于讲解）
- 剪辑掉录制错误的部分
